---
description: 按固定标准审代码：bug、边界条件、可维护性、安全风险、命名混乱
---

对当前改动进行代码审阅。按以下维度逐一检查，输出按严重度分级（致命/严重/建议），附带具体文件和行号：

1. **Bug / 逻辑错误**：条件判断是否有误、循环边界是否正确、None/空值处理
2. **边界条件**：空输入、超大输入、并发写入、文件不存在
3. **可维护性**：重复代码、函数过长（>50行需关注）、嵌套过深
4. **安全风险**：SQL 注入、路径遍历、未授权访问、敏感信息泄露
5. **命名一致性**：变量/函数名是否与项目已有风格一致

只审阅 `git diff` 中实际改动的行，不审阅未改动代码。如果没问题，直接说"未发现问题"。
