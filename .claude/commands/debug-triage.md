---
description: 出错时先收集报错、环境、最近改动，再定位可能原因
---

按以下流程排查问题，不盲目改代码：

1. **收集信息**：
   - 完整报错信息和堆栈
   - Python 版本：`python --version`
   - 操作系统和环境变量
   - 最近 git 改动：`git diff HEAD~1 --stat` 和 `git log --oneline -5`

2. **定位原因**：
   - 从堆栈最底层（根因）开始分析
   - 检查是否与最近的代码改动相关
   - 检查是否有依赖缺失或版本不匹配

3. **提出修复方案**：
   - 先说明原因，再给修复代码
   - 如果涉及数据，先提醒备份
   - 修复后说明如何验证

在未确认根因之前，不要直接修改代码。
